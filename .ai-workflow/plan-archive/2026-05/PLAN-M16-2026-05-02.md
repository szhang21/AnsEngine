## PlanningDecisions

- M16 直接迁移 schema，不做双轨兼容。
- 旧 `version: "1.0"` 扁平对象格式加载失败。
- Scene JSON version 升级到 `2.0`。
- Object component schema 使用数组：
  - `components: [...]`
  - 每个 component 用 Pascal 简名 `type`。
  - 首批 type 固定为 `Transform` 与 `MeshRenderer`。
- Component payload 使用扁平字段，不使用 `value` 包装。
- `Transform` 是必需组件。
- `MeshRenderer` 是可选组件。
- 同一 object 内重复 component type 加载失败。
- 未知 component type 加载失败。
- Camera 暂不组件化，继续保持 scene-level camera。
- Editor 用户界面开始显式展示组件分组，而不是继续把 mesh/material/transform 混在一个扁平面板里。
- SceneData 编辑 API 迁移为 component API，不继续把旧 `UpdateObjectTransform` / `UpdateObjectResources` 作为主 API。

## SchemaDesign

示例 JSON：

```json
{
  "version": "2.0",
  "scene": {
    "id": "sample-scene",
    "name": "Sample Scene",
    "camera": {
      "position": { "x": 0, "y": 0.25, "z": 2.2 },
      "target": { "x": 0, "y": 0, "z": 0 },
      "fieldOfViewRadians": 1.0471976
    },
    "objects": [
      {
        "id": "cube-main",
        "name": "Cube Main",
        "components": [
          {
            "type": "Transform",
            "position": { "x": 0, "y": 0, "z": 0 },
            "rotation": { "x": 0, "y": 0, "z": 0, "w": 1 },
            "scale": { "x": 1, "y": 1, "z": 1 }
          },
          {
            "type": "MeshRenderer",
            "mesh": "mesh://cube",
            "material": "material://default"
          }
        ]
      }
    ]
  }
}
```

File model guidance:

- `SceneFileObjectDefinition`
  - `Id`
  - `Name`
  - `Components`
- Suggested new file model records:
  - `SceneFileComponentDefinition`
  - `SceneFileTransformComponentDefinition`
  - `SceneFileMeshRendererComponentDefinition`
- JSON serializer must choose concrete component payload by `type`.
- `SceneFileTransformDefinition` may be reused or replaced by `SceneFileTransformComponentDefinition`; Dispatch must choose one clean path and avoid duplicate semantic types unless there is a compatibility reason.

## NormalizedModelDesign

- `SceneObjectDescription` should no longer directly hold `Mesh` / `Material` / `LocalTransform`.
- It should hold normalized component descriptions, for example:
  - `SceneComponentDescription`
  - `SceneTransformComponentDescription`
  - `SceneMeshRendererComponentDescription`
- Transform component description contains normalized finite position / rotation / scale.
- MeshRenderer component description contains normalized `SceneMeshRef` and `SceneMaterialRef`.
- `SceneDescription` and camera model remain otherwise unchanged.
- `Engine.SceneData` must not reference `Engine.Scene` runtime component types.

## RuntimeBridgeDesign

- `RuntimeScene.LoadFromDescription(...)` should iterate normalized component descriptions.
- For `Transform`, create `SceneTransformComponent`.
- For `MeshRenderer`, create `SceneMeshRendererComponent`.
- If object has Transform only:
  - runtime object is created;
  - runtime snapshot includes object and transform;
  - `HasMeshRenderer=false`;
  - render frame excludes it.
- If object has Transform + MeshRenderer:
  - runtime object is created with both components;
  - render frame outputs item as before.
- Runtime bridge must not read JSON or know file schema details.
- `BuildRenderFrame()` remains a pure read of runtime state.
- M15 update behavior remains unchanged: it rotates first object that has both Transform and MeshRenderer.

## ValidationRules

- Scene document `version` must be `"2.0"`.
- Missing `scene`, missing scene id, missing objects collection still fail as before.
- Each object id must be non-empty and unique.
- Each object must have non-null, non-empty `components`.
- Each object must have exactly one `Transform` component.
- Each object may have zero or one `MeshRenderer` component.
- Duplicate component type fails.
- Unknown component type fails.
- `Transform` position / rotation / scale must be finite.
- `MeshRenderer.mesh` must be present and valid scene reference.
- `MeshRenderer.material` defaults to `material://default` when null/blank, otherwise must be valid scene reference.

## EditorDesign

- Inspector should display component groups:
  - `Object`: object id/name.
  - `Transform`: position/rotation/scale.
  - `MeshRenderer`: mesh/material or a clear no-MeshRenderer state.
- Editor default Add Object still creates both `Transform` and `MeshRenderer`, keeping newly added objects visible.
- Editor must be able to open/select/save Transform-only objects without auto-adding MeshRenderer.
- Editor should not silently repair missing optional components during save.
- SceneData editing moves to component API:
  - Add/update/remove object component operations.
  - Existing object id/name operations may remain object-level.
  - Old transform/resources methods should not remain the primary path after M16.
- Dirty/save/reload semantics must remain stable.

## Risks

- Risk: high
  - 描述：M16 是 breaking scene schema migration，会影响 SceneData、Scene、Editor、App sample 和大量测试。
  - 规避：串行拆卡，先稳定 SceneData schema 与 normalized model，再迁移 runtime 和 editor。
- Risk: medium
  - 描述：component array 反序列化可能引入多态 serializer 复杂度。
  - 规避：首批仅支持两个 Pascal type，未知 type fail fast。
- Risk: medium
  - 描述：Editor component API 迁移可能扩大到 `Engine.Editor` 与 `Engine.Editor.App` 两个主模块。
  - 规避：Dispatch 必须按主模块边界拆卡或明确主模块，不把过宽跨模块改动塞入单卡。
- Risk: low
  - 描述：Transform-only object 可能被误认为渲染异常。
  - 规避：测试和 Inspector 明确显示无 MeshRenderer 状态，render frame 不输出该 object 是预期行为。
