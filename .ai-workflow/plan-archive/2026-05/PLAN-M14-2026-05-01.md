# Plan Archive: PLAN-M14-2026-05-01

## GoalSummary

- 本轮目标：M14 将 `Engine.Scene` 从“直接把 `SceneDescription` 转成 `SceneRenderFrame` 的服务”推进为“持有运行时对象和基础组件的 Runtime Scene”。
- 成功标准（可验证）：
  - `Engine.Scene` 内新增轻量 runtime object/component model。
  - `SceneGraphService.LoadSceneDescription(SceneDescription)` 创建 runtime objects，而不是只填充 render item list。
  - `SceneGraphService.BuildRenderFrame()` 从 runtime objects/components 生成 `SceneRenderFrame`。
  - 现有 `Engine.App -> SceneData -> Scene -> Render` 链路不退化。
  - `Engine.Render` 仍只消费 `Engine.Contracts.SceneRenderFrame`，不感知 runtime object/component。
  - M14 不做脚本、物理、动画、Editor Viewport、Gizmo 或 Play Mode。

## Context

- M13 收尾后，GUI 编辑器已经成为可操作入口，但最终目标不只是编辑器，还包括脚本、物理、动画等完整引擎系统。
- 当前 `SceneGraphService` 仍主要是 `SceneDescription -> SceneRenderFrame` 的转换/提供器，不是真正的 runtime scene owner。
- `engine-scene` 边界已经把 Entity、Component、Transform 和场景查询列为 `Engine.Scene` 职责，因此 M14 不创建新模块，先在 `Engine.Scene` 内落地轻量 GameObject/Component 风格底座。
- M15 将在 M14 基础上继续做 Runtime Update Pipeline，M16 再考虑 Component Serialization Bridge / SceneData schema 扩展。

## RuntimeModel

- `SceneRuntimeObject`
  - 每个 `SceneObjectDescription` 映射成一个 runtime object。
  - 建议属性：`NodeId`、`ObjectId`、`ObjectName`、`Transform`、`MeshRenderer`。
  - `NodeId` 继续用于 `SceneRenderItem.NodeId`。
  - `ObjectId` 来自 `SceneObjectDescription.ObjectId`，后续脚本/物理/动画引用对象时优先使用。
  - 所有 object 都有 `Transform`。
  - M14 从现有 scene description 映射时默认每个 object 都有 `MeshRenderer`。
  - 暂不做 parent/children 层级。

- `SceneTransformComponent`
  - 保存 local position/rotation/scale。
  - 提供转换到 `Engine.Contracts.SceneTransform` 的方法。
  - 提供最小本地 transform 更新方法，供测试和未来系统使用。
  - 不计算 world transform，不处理 parent hierarchy。

- `SceneMeshRendererComponent`
  - 持有 `SceneMeshRef` 与 `SceneMaterialRef`。
  - 不加载 mesh 文件，不检查资源存在性，不依赖 `Engine.Asset`。

- `SceneCameraRuntimeState`
  - 从 `SceneCameraDescription` 映射。
  - 缺省时沿用当前默认相机语义。
  - 提供 `BuildCamera(aspectRatio, nearPlane, farPlane)` 或等价能力生成 `SceneCamera`。
  - M14 暂不把 camera 做成挂在 object 上的 component。

- `RuntimeScene`
  - 是 `Engine.Scene` 内部 runtime scene owner。
  - 持有 runtime objects 和 camera runtime state。
  - 不读 JSON，不知道文件路径，不知道 render backend。
  - 提供 load、build render frame、create snapshot 的内部能力。

## SceneGraphServiceChanges

- `SceneGraphService` 继续实现 `ISceneRenderContractProvider.BuildRenderFrame()`。
- 内部状态目标：
  - 持有一个 `RuntimeScene`。
  - 保留 `mFrameNumber`。
  - `NodeCount` 返回 runtime object count。
- `LoadSceneDescription(SceneDescription)`：
  - 清空旧 runtime state。
  - 创建新的 runtime objects。
  - 重置 frame number。
- `BuildRenderFrame()`：
  - 从 runtime objects 中筛选带 `Transform + MeshRenderer` 的对象输出 `SceneRenderItem`。
  - `SceneRenderItem.NodeId = SceneRuntimeObject.NodeId`。
  - mesh/material/transform 来自 components。
  - camera 来自 runtime camera state。
  - frame number 保持自增。
- 旧的 `AddRootNode()` demo path 可以保留，但必须与 scene description runtime path 分开，不污染加载 scene 文件后的稳定输出。

## PublicQuerySurface

- 为测试和未来系统提供只读查询，不暴露可变集合。
- 建议在 `SceneGraphService` 增加：
  - `CreateRuntimeSnapshot()`
  - `FindObject(string objectId)`
- Snapshot 建议：
  - `RuntimeSceneSnapshot`
  - `SceneRuntimeObjectSnapshot`
- Snapshot 至少包含：
  - node id
  - object id
  - object name
  - local transform
  - mesh/material 或 `HasMeshRenderer` + refs
  - camera snapshot
- Snapshot 是只读值对象，外部不能通过 snapshot 修改 runtime state。

## Risks

- 高风险：把 runtime object/component 暴露成跨模块公共事实，导致 Render/App/Editor 直接依赖 Scene 内部类型。
  - 规避策略：Render 继续只消费 `SceneRenderFrame`，测试用 snapshot 而不是可变内部集合。
- 中风险：`SceneGraphService` 同时承担 facade 和 runtime owner，后续变复杂。
  - 规避策略：M14 允许过渡；后续 M15/M16 如复杂度上升再拆 `RuntimeSceneService` 或独立模块。
- 中风险：旧 demo dynamic path 与 scene description path 语义混杂。
  - 规避策略：明确 `LoadSceneDescription` 后走 runtime scene 稳定输出，`AddRootNode` 保留为 legacy/demo path。
- 低风险：M14 不扩展 scene JSON，component 序列化能力需要 M16 再补。
  - 规避策略：M14 只映射已有 `SceneDescription` 字段。

## Assumptions
