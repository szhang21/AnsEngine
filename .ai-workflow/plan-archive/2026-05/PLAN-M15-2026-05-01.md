# Plan Archive: PLAN-M15-2026-05-01

## GoalSummary

- 本轮目标：在 M14 的 runtime object/component foundation 上建立核心 `Update` 生命周期，让 `Engine.App` 每帧在 Render 前驱动 `Engine.Scene` runtime。
- 成功标准（可验证）：
  - `Engine.Scene` 提供 Scene 自有 update context 与 public update 入口。
  - `RuntimeScene` 维护 update frame count 与 accumulated update seconds。
  - `Engine.App` 主循环在 `RenderFrame()` 前调用 runtime update。
  - update 后 runtime transform 变化能通过 `CreateRuntimeSnapshot()` 与 `BuildRenderFrame()` 被观察到。
  - `Engine.Scene` 不新增 `Engine.Platform`、`Engine.Render`、`Engine.App` 依赖。
  - M15 不做脚本系统、物理系统、动画系统、Editor Play Mode、SceneData schema 扩展或 Render contract 变更。

## Context

- M14 已完成轻量 runtime object/component foundation：`SceneGraphService` 持有 `RuntimeScene`，`LoadSceneDescription` 映射 runtime objects/components，`BuildRenderFrame()` 从 runtime state 输出 `SceneRenderFrame`。
- 当前 `Engine.App` 主循环已有 `IInputService` 与 `ITimeService`，但 scene runtime 只在启动时初始化一次，没有 per-frame update 入口。
- M15 是 runtime/game loop 地基，不应被建模成某个 component 或 system；脚本、物理、动画未来都应挂在同一个 runtime update 节拍之后演进。

## Scope

- In scope:
  - Scene 自有 update context。
  - RuntimeScene 单阶段 update lifecycle。
  - Update 统计与 snapshot 诊断字段。
  - 默认启用的最小旋转 smoke behavior。
  - App 主循环 Render 前 update 接线。
  - Scene/App/Render 边界与回归测试。
- Out of scope:
  - 脚本系统。
  - 物理系统。
  - 动画系统。
  - Editor Play Mode。
  - SceneData JSON schema 扩展。
  - `Engine.Contracts.SceneRenderFrame` 或 Render public contract 变更。
  - 正式 component/system 抽象。

## PlanningDecisions

- Update 是 runtime/game loop 的核心机制，不建模为 component 或 system。
- M15 只做单阶段 `Update`，不做 `PreUpdate` / `PostUpdate` / `FixedUpdate`。
- `Engine.Scene` 定义自有 update context，避免新增 `Engine.Scene -> Engine.Platform` 依赖。
- 不把 update context 放入 `Engine.Contracts`，避免过早扩大公共契约面。
- App adapter 负责把 `TimeSnapshot` / `InputSnapshot` 翻译为 Scene update context。
- 默认启用最小旋转 smoke behavior：update 时旋转第一个带 `Transform` 与 `MeshRenderer` 的 runtime object。
- 旋转行为只用于证明 runtime update 正在推进，不作为正式 component/system 设计。
- `SceneRenderFrame.FrameNumber` 与 runtime update frame count 语义分离：
  - render frame number 表示 render submission 生成次数。
  - update frame count 表示 runtime simulation 步进次数。
- `BuildRenderFrame()` 必须是读取当前 runtime state，不得隐式推进 update。

## CodeDesignNotes

- 建议新增 `SceneUpdateContext`：
  - 位置：`Engine.Scene`。
  - 形状：`public readonly record struct SceneUpdateContext(double DeltaSeconds, double TotalSeconds, bool AnyInputDetected)`。
  - `DeltaSeconds < 0` 抛 `ArgumentOutOfRangeException`。
  - `DeltaSeconds == 0` 合法，记录一次 update，但不推进旋转角度。
- 建议扩展 `SceneGraphService`：
  - 新增 `public void UpdateRuntime(SceneUpdateContext context)`。
  - 只做参数转发与 runtime ownership 边界，不直接写 transform 细节。
- 建议扩展 `RuntimeScene`：
  - 新增 `UpdateFrameCount`。
  - 新增 `AccumulatedUpdateSeconds`。
  - 新增 `Update(SceneUpdateContext context)`。
  - `Clear()` / `LoadFromDescription(...)` 重置 update 统计。
  - update 不创建/删除对象，不改 mesh/material/camera，不做资源加载。
- 最小旋转 smoke behavior：
  - 每次 update 找到第一个同时具备 `Transform` 与 `MeshRenderer` 的 object。
  - 仅修改该 object 的 `SceneTransformComponent.LocalRotation`。
  - 绕 Y 轴以固定角速度推进，建议 `MathF.PI * 0.5f` radians/sec。
  - 保持 position/scale 不变。
  - 没有可渲染对象时不报错，只更新诊断状态。
- Snapshot 扩展：
  - `RuntimeSceneSnapshot` 增加 `UpdateFrameCount` 与 `AccumulatedUpdateSeconds`。
  - 不暴露 mutable runtime object/component。
  - `SceneRuntimeObjectSnapshot` 暂不新增字段，通过现有 `LocalTransform` 观察 rotation。
- App integration：
  - `ISceneRuntime` 保留 `InitializeScene(SceneDescription sceneDescription)`。
  - `ISceneRuntime` 新增 `Update(SceneUpdateContext context)` 或等价入口。
  - `SceneRuntimeAdapter.Update` 调用 `SceneGraphService.UpdateRuntime`。
  - `ApplicationHost.Run()` 每帧顺序固定为：
    1. `mWindowService.ProcessEvents()`
    2. `var input = mInputService.GetSnapshot()`
    3. `var time = mTimeService.Current`
    4. `mSceneRuntime.Update(new SceneUpdateContext(time.DeltaSeconds, time.TotalSeconds, input.AnyInputDetected))`
    5. `mRenderer.RenderFrame()`
    6. `mWindowService.Present()`
- Design constraints:
  - 不要让 `Engine.Scene` 引用 `Engine.Platform.TimeSnapshot` 或 `InputSnapshot`。
  - 不要在 `BuildRenderFrame()` 内推进时间或旋转。
  - 不要新增正式 component/system 抽象来承载旋转行为。
  - 不要扩 SceneData schema，也不要在 JSON 里加 rotate 字段。
  - 不要新增 `Engine.Scene -> Engine.App`、`Engine.Scene -> Engine.Render`、`Engine.Scene -> Engine.Platform` 依赖。
  - 遵循 C# 命名规范：private fields 用 `mCamelCase`，static fields 用 `sCamelCase`，const 用 `kCamelCase`。
  - 旋转实现必须 deterministic：只依赖传入的 `DeltaSeconds`，不读 wall clock，不读 renderer frame number。
  - 更新 transform 时复用 `SceneTransformComponent.SetLocalTransform(...)`。

## Milestones

- M15.1: Runtime update context 与统计地基
  - 状态：Planned
  - 主模块：`Engine.Scene`
  - 优先级：P0
  - 交付结果：
    - 新增 Scene 自有 update context。
    - `SceneGraphService` 暴露 update runtime 入口。
    - `RuntimeScene` 支持 update frame count 与 accumulated update seconds。
    - `Clear()` / `LoadFromDescription(...)` 重置 update 统计。
  - 验收口径：
    - update 后统计递增。
    - zero delta 合法且统计递增。
    - negative delta 抛出可诊断异常。
    - 空 scene update 不崩溃。
    - `Engine.Scene` 不新增禁止依赖。

- M15.2: 默认最小旋转 smoke behavior
  - 状态：Planned
  - 主模块：`Engine.Scene`
  - 优先级：P1
  - 交付结果：
    - `RuntimeScene.Update` 旋转第一个可渲染 runtime object。
    - update 后 snapshot 与 render frame 均可观察 rotation 变化。
  - 验收口径：
    - 有可渲染 object 时 update 改变 rotation。
    - position/scale 不变。
    - `BuildRenderFrame()` 输出 update 后 rotation。
    - 没有可渲染 object 时不抛错。

- M15.3: App 主循环接入 runtime update
  - 状态：Planned
  - 主模块：`Engine.App`
  - 优先级：P1
  - 交付结果：
    - `ISceneRuntime` 扩展 update 入口。
    - `SceneRuntimeAdapter` 翻译 App time/input 到 Scene update context。
    - `ApplicationHost.Run()` 每帧在 render 前调用 scene update。
  - 验收口径：
    - 成功路径至少调用一次 update。
    - update 发生在 render 前。
    - loader failure 不调用 initialize/update/render。
    - render failure 仍 request close、shutdown、dispose。

- M15.4: Snapshot 诊断字段与边界测试
  - 状态：Planned

    - 补充 Scene/Render/App 边界测试。
  - 验收口径：
    - snapshot 不暴露 mutable runtime state。
    - update 诊断字段与 runtime update 状态一致。
    - `Engine.Render` 不引用 update context 或 runtime scene 类型。

## TestPlan

- Scene tests:
  - `UpdateRuntime` 调用后 `UpdateFrameCount` 从 0 变 1。
  - 多次 update 后 `AccumulatedUpdateSeconds` 为 delta 累加值。
  - `DeltaSeconds == 0` 增加 update count，但 rotation 不变。
  - `DeltaSeconds < 0` 抛 `ArgumentOutOfRangeException`。
  - 有可渲染 object 时，update 后 snapshot 的 `LocalTransform.Rotation` 改变。
  - update 后 `BuildRenderFrame()` 输出的 item rotation 与 runtime snapshot 一致。
  - 空 scene update 不崩溃，诊断状态仍更新。
  - `LoadSceneDescription` 后 update 统计重置。
- App tests:
  - `ApplicationHost.Run()` 成功路径至少调用一次 `sceneRuntime.Update`。
  - update 发生在 `RenderFrame()` 前。
  - adapter 正确传递 `DeltaSeconds`、`TotalSeconds`、`AnyInputDetected`。
  - loader failure 不调用 initialize/update/render。
  - render failure 仍 request close、shutdown、dispose。
- Boundary tests:
  - `Engine.Scene` 不引用 `Engine.Platform`、`Engine.Render`、`Engine.App`。
  - `Engine.Render` 不引用 `SceneUpdateContext` 或 runtime scene 类型。
- Validation commands:
  - `dotnet build AnsEngine.sln --nologo -v minimal`
  - `dotnet test AnsEngine.sln --no-restore --nologo -v minimal`

## Risks

- Risk: medium
  - 描述：默认旋转 smoke behavior 可能被误解为正式动画/脚本能力。
  - 规避：任务卡必须明确它只是 M15 update 可见性验证，不扩 SceneData schema，不新增正式 component/system。
- Risk: medium
  - 描述：如果 Scene 直接依赖 Platform 时间/输入类型，会破坏模块边界。
  - 规避：Scene 自有 context，App adapter 翻译。
- Risk: low
  - 描述：render frame number 与 update frame count 混用会导致测试和后续系统语义不清。
  - 规避：两个计数保持独立，分别测试。
