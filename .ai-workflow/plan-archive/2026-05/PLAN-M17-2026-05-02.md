## PlanningDecisions

- M17 目标是 Scripting foundation，不是完整 scripting MVP。
- 新建独立 `Engine.Scripting` 模块。
- 依赖方向：`Engine.Scripting -> Engine.Scene`。
- 禁止 `Engine.Scene -> Engine.Scripting`。
- 第一版 C# script 只通过内置注册表加载。
- 不加载外部程序集，不编译脚本源码。
- Script component 使用：
  - `type: "Script"`
  - `scriptId`
  - `properties`
- `properties` 第一版只支持 number / bool / string。
- 同一对象允许多个 Script component，按 scene 文件顺序 initialize 和 update。
- 未注册 `scriptId` 绑定失败。
- 属性类型不符合脚本期望时绑定失败。
- Script `Initialize` / `Update` 抛异常时 fail fast，App run 返回错误并正常 shutdown/dispose。
- 第一版脚本只能访问自身对象 Transform。
- 不允许脚本查询或修改其他对象。
- M15 默认旋转 smoke behavior 在 M17 中移除或禁用；可见 runtime 行为由内置 `RotateSelf` script 接管。
- Editor 暂不支持编辑 Script component，但打开/保存必须保留 Script component。

## ScriptingDesign

- Suggested public types in `Engine.Scripting`:
  - `IScriptBehavior`
  - `ScriptContext`
  - `ScriptRegistry`
  - `ScriptRuntime`
  - `ScriptBindingResult` or equivalent failure result
  - `ScriptUpdateResult` or equivalent failure result
- Suggested lifecycle:
  - `Initialize(ScriptContext context)` exactly once after scene load / script binding.
  - `Update(ScriptContext context)` once per App runtime update frame.
- Suggested registry behavior:
  - Register script factory by `scriptId`.
  - Duplicate `scriptId` registration fails.
  - Missing `scriptId` creation fails.
  - Registered factory creates a new script instance per Script component.
- Suggested context contents:
  - object id.
  - object name.
  - delta seconds.
  - total seconds.
  - read-only properties map.
  - self Transform accessor.
- Suggested property value model:
  - `double`
  - `bool`
  - `string`
  - No arrays/objects/Vector3 in M17.
- Built-in sample:
  - `RotateSelf`
  - property: `speedRadiansPerSecond`
  - reads `speedRadiansPerSecond` as number.
  - update multiplies self rotation by Y-axis delta rotation.

## SceneDataDesign

- Add `Script` component to SceneData component schema:

```json
{
  "type": "Script",
  "scriptId": "RotateSelf",
  "properties": {
    "speedRadiansPerSecond": 1.5708
  }
}
```

- `Script` component is repeatable.
- `scriptId` must be non-empty.
- `properties` may be empty.
- `properties` values must be number / bool / string only.
- Unknown property names are allowed at SceneData level; script binding decides whether required properties exist and have expected types.
- Normalized model should include `SceneScriptComponentDescription`.
- Script component order must be preserved from scene file to normalized description.
- SceneData must not depend on `Engine.Scripting` types.

## SceneBridgeDesign

- `Engine.Scene` exposes narrow script access API, for example:
  - bind script to object id.
  - get object id/name.
  - get/set self local transform.
- API must not expose mutable runtime object/component collections.
- API must not allow querying arbitrary objects in M17.
- Transform-only objects can host scripts.
- Objects without Transform are not expected because M16 requires Transform.
- Scene script bridge should keep `BuildRenderFrame()` as a pure read of current runtime state.

## AppIntegrationDesign

- `RuntimeBootstrap` composes:
  - `SceneGraphService`
  - `ScriptRegistry`
  - `ScriptRuntime`
  - built-in `RotateSelf` registration
- Scene load flow:
  1. Load SceneData.
  2. Initialize Scene runtime.
  3. Bind Script components to runtime objects.
  4. Initialize script instances.
- Per-frame flow:
  1. Process events.
  2. Read input.
  3. Read time.
  4. Scene base update.
  5. Script update.
  6. Render frame.
  7. Present.
- Script binding/update failure:
  - App returns error.
  - Window close/shutdown/dispose behavior remains stable.

## EditorPreservationDesign

- Editor is not required to add or edit Script component in M17.
- Editor must preserve Script components during open/save/save-as.
- Inspector may ignore Script components in M17.
- Save/reload must not drop Script component order or properties.
- Full Script component editor UI is deferred to a later milestone.
