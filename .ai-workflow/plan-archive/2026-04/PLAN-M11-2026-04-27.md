  - M11 不新增 `Engine.Editor`、`Engine.Tools` 或类似编辑器宿主模块；新宿主模块留到 M12。

## ModuleBoundaries

- `Engine.SceneData`
  - 职责：场景文件模型、JSON 读写、文档级编辑操作、默认值规范化、校验与显式失败语义。
  - 非职责：GUI 状态、选中态、Gizmo、Undo/Redo、运行时对象、world transform 缓存、asset catalog、OBJ 导入、GPU 资源。
  - 允许依赖：`Engine.Contracts`、`.NET` 标准库、`System.Text.Json`。
  - 禁止依赖：`Engine.Scene`、`Engine.App`、`Engine.Asset`、`Engine.Render`、`Engine.Platform`。

- `Engine.Editor` / `Engine.Tools`
  - M11 不创建该类新模块。
  - 规划定位：M12 才开始承载编辑器宿主、工具入口、编辑器状态或 UI 编排。
  - M11 只为这些未来模块提供可消费的 `Engine.SceneData` 文档能力。

- `Engine.Scene`
  - 职责：消费规范化 `SceneDescription`，初始化运行时场景并输出 `SceneRenderFrame`。
  - M11 默认不新增职责；不得读取/保存 JSON，不得执行编辑器文档操作。
  - 允许依赖保持：`Engine.Core`、`Engine.Contracts`、`Engine.SceneData`。
  - 禁止新增依赖：`Engine.Asset`、`Engine.Render`、`Engine.App`。

- `Engine.App`
  - 职责：组合根装配、场景路径选择、运行时启动失败收口。
  - M11 可选增加最小非 GUI 工具入口装配，但不能承载场景编辑业务逻辑。
  - 允许依赖保持：`Engine.Core`、`Engine.Platform`、`Engine.Scene`、`Engine.SceneData`、`Engine.Asset`、`Engine.Render`、`Engine.Contracts`。
  - 禁止：在 App 中实现 JSON 修改、对象增删改、资源解析或渲染细节。

- `Engine.Contracts`
  - 职责：跨模块稳定契约，如 `SceneMeshRef`、`SceneMaterialRef`。
  - M11 默认不修改；只有 SceneData 编辑结果确实需要共享契约时才允许小步扩展。
  - 禁止：放入文档编辑流程、JSON IO 或运行时状态。

- `Engine.Asset`
  - M11 默认不改。
  - 职责仍是 mesh catalog、磁盘资源导入、CPU asset cache。
  - 禁止参与 SceneData 文档校验中的真实资源存在性检查；M11 只校验引用格式，不校验资产是否真实存在。

- `Engine.Render`
  - M11 默认不改。
  - 继续只消费 `SceneRenderFrame` 和 mesh provider，不感知 JSON 或编辑器文档模型。

## DependencyRules

- 保持依赖主线：`Engine.SceneData -> Engine.Contracts -> Engine.Core`。
- 保持运行数据流：`Engine.App -> Engine.SceneData -> Engine.Scene -> Engine.Contracts -> Engine.Render`；这是数据流与装配流，不代表反向项目依赖。
- 禁止形成依赖环：`Engine.SceneData` 永远不能依赖 `Scene/App/Asset/Render/Platform`。
- 禁止跨层校验：`SceneData` 不检查 mesh 文件是否存在；`Asset` 不读取 scene JSON；`Render` 不感知 scene file DTO。
- 新增文件时必须更新至少一个 `.ai-workflow/boundaries/*.md`，并写入 `Boundary Change Log`。

## PlanningDecisions

- 默认方向：M11 在 `Engine.SceneData` 内做编辑器数据能力底座，不新增编辑器宿主模块，不做 GUI 编辑器。
- 编辑器状态边界：`SceneData` 不保存 selection、Gizmo、Undo/Redo、viewport camera 或 UI 状态。
- 保存对象：优先保存 `SceneFileDocument` 文件描述层，不直接保存规范化 `SceneDescription`。
- 运行时入口：`ISceneDescriptionLoader` 继续服务运行时加载，新的文档读写入口服务编辑/工具链。
- 校验范围：M11 校验 schema、必填字段、重复对象 id、引用格式与 transform 数值；不校验 mesh/material 资源是否真实存在。
- 依赖规则：`SceneData -> Contracts` 是唯一项目依赖方向，不能为了编辑便利引入 `Scene/App/Asset/Render`。
- 后续路线：M12 新增最小编辑器宿主模块或工具入口，M13 再做可视编辑器。

## Risks

- 高风险：把编辑器状态塞进 `SceneData`。
  - 规避策略：M11 只做文档数据，不做 selection、Gizmo、Undo/Redo、viewport camera。
- 中风险：文档保存与运行时加载规则分叉。
  - 规避策略：抽出或复用校验/规范化规则，并强制 load-save-load 测试。
- 中风险：App 工具入口变成业务逻辑堆积点。
  - 规避策略：App 只装配和调用 SceneData 服务，不实现编辑规则。
- 低风险：当前目标框架 `net7.0` 已 EOL。
  - 规避策略：记录为独立技术债，不混入 M11 功能范围。
- 低风险：当前环境 `git status` 受 Xcode command line tools 缺失影响不可用。
  - 规避策略：不影响计划落盘；后续提交/分支动作需先修本机 Git 环境。
