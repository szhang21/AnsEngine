# Plan Archive: PLAN-M10-2026-04-25

## GoalSummary

- 本轮目标：把当前“场景内容主要硬编码在 `SceneGraphService`”的状态，升级为“`JSON 场景文件 -> SceneData 描述 -> Scene 运行时对象 -> Render 现有链路`”。
- 成功标准（可验证）：
  - 新增独立 `Engine.SceneData` 模块，负责场景 JSON 描述类型、解析、校验和默认值规范化。
  - `Engine.Scene` 不直接做文件 IO、JSON 解析、asset catalog 解析或 mesh 导入。
  - `Engine.App` 只负责选择场景文件并装配 loader，不承载场景业务逻辑。
  - `SceneData` 中对象实例以 `LocalTransform` 描述，mesh 继续保持模型空间几何，Render 负责消费 `mesh + transform + camera` 生成提交参数。
  - 至少一个样例场景文件可驱动现有 M9 真实 mesh 资产链路运行，且修改场景内容无需改 C# 代码。

## PlanningDecisions

- 默认方向：新增 `Engine.SceneData`，不把场景 JSON 解析塞进 `Engine.Scene` 或 `Engine.App`。
- 默认依赖：`Engine.SceneData -> Engine.Contracts`，不新增 `Engine.Core` 业务桥接职责。
- 数据分层：`SceneData` 至少分为“文件描述层”和“规范化场景层”，不只保留一套原始 JSON DTO。
- Transform 语义：对象实例使用 `LocalTransform`；M10 无层级时它等价 world transform，M11 加层级后由 `Scene` 负责 `Local -> World`。
- JSON 第一版范围：对象列表、`meshId`、`materialId`、`LocalTransform`、单相机；不做层级、Prefab、灯光、glTF 场景导入。
- 默认值：
  - 缺省 `materialId = material://default`
  - 缺省 `LocalTransform = Identity`
  - 缺省相机时生成引擎默认相机
  - `meshId` 必填
  - 对象 `Id` 必填且唯一
- 编辑器关系：未来编辑器优先对接 `Engine.SceneData`，但 `SceneData` 不是编辑器状态模型，不承载选中、Gizmo、Undo 栈等状态。

  - `SceneFileObjectDefinition`
    - `Id`

    - `Mesh`
    - `Material`
    - `Transform`
  - `SceneFileCameraDefinition`
    - `Position`
    - `Target`
    - `FieldOfViewRadians`
  - `SceneFileTransformDefinition`
    - `Position`
    - `Rotation`
    - `Scale`

- 规范化场景层：
  - `SceneDescription`
    - `SceneId`
    - `SceneName`
    - `Camera`
    - `Objects`
  - `SceneObjectDescription`
    - `ObjectId`
    - `ObjectName`
    - `Mesh`
    - `Material`
    - `LocalTransform`
  - `SceneCameraDescription`
    - `Position`
    - `Target`
    - `FieldOfViewRadians`
  - `SceneTransformDescription`
    - `Position`
    - `Rotation`
    - `Scale`

- 加载与失败语义：
  - `ISceneDescriptionLoader`
    - `Load(string sceneFilePath)`
  - `SceneDescriptionLoadResult`
    - `IsSuccess`
    - `Scene`
    - `Failure`
  - `SceneDescriptionLoadFailure`
    - `Kind`
    - `Message`
    - `Path`
    - `LineNumber`（可选）
  - `SceneDescriptionLoadFailureKind`
    - `NotFound`
    - `InvalidJson`
    - `MissingRequiredField`
    - `DuplicateObjectId`
    - `InvalidReference`
    - `InvalidValue`

- 模块内明确不包含：
  - Scene 运行时对象
  - world transform 缓存
  - mesh 顶点数据或 GPU 资源
  - asset catalog / OBJ 导入
  - 编辑器状态（选中、Gizmo、Undo 栈等）

    - `M10.3`：Scene/App 接线与样例场景

- 推荐波次：
  - Wave1：Contracts/SceneData 边界与模型卡
  - Wave2：SceneData loader 与失败语义卡
  - Wave3：Scene 初始化入口与 App 装配卡
  - Wave4：QA 与归档收口卡
- 主模块边界：
  - `Engine.SceneData`：场景文档模型、JSON 解析、校验、默认值规范化
  - `Engine.Scene`：运行时对象、场景状态、渲染输入输出
  - `Engine.App`：场景文件选择、loader 装配、主循环接线
  - `Engine.Contracts`：稳定资源引用与跨模块共享语义
- 关键门禁：
  - 不允许 `Engine.Scene` 直接解析 JSON 或做文件 IO 流程编排
  - 不允许 `Engine.App` 承载场景业务模型
  - 不允许 `SceneData` 依赖 `Scene/Asset/Render`
  - 不允许把 `LocalTransform` 语义写死成 world-only
