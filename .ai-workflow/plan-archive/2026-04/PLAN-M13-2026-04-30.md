# Plan Archive: PLAN-M13-2026-04-30

## GoalSummary

- 本轮目标：M13 将 M12 的 `Engine.Editor` headless core 接成真正可运行、可点击、可保存的最小 GUI 场景编辑器。
- 成功标准（可验证）：
  - 新增独立可执行编辑器宿主 `Engine.Editor.App`。
  - 运行 `dotnet run --project src/Engine.Editor.App/Engine.Editor.App.csproj` 能启动编辑器窗口。
  - 默认打开源码目录下的 sample scene：`src/Engine.App/SampleScenes/default.scene.json`。
  - 用户可以通过 GUI 在 Hierarchy 选择对象，在 Inspector 修改对象字段，并 Save 写回 `.scene.json`。
  - 保存后运行 `Engine.App` 可读取修改后的场景。
  - `Engine.Editor` 保持 headless core，不引入 GUI、OpenTK、ImGui、窗口或输入依赖。
  - `Engine.App` 继续作为运行时入口，不接入编辑器 GUI。

## Context

- M12 已关闭，`Engine.Editor` 已提供 `SceneEditorSession`，支持打开/关闭、状态查询、selection、对象编辑、保存、另存为与 reload/normalize 验证。
- M13 的核心分界：M12 是 API 能编辑场景；M13 是人能通过 GUI 编辑场景。
- M13 默认技术路线：`OpenTK + ImGui.NET`。
- M13 不做完整编辑器，只做最小可用 GUI 工作流。

## ModuleBoundaries

- `Engine.Editor.App`
  - 职责：GUI 宿主、窗口主循环、工具栏、Hierarchy、Inspector、状态栏、用户输入到 `SceneEditorSession` 的编排。
  - 非职责：底层 JSON 编辑规则、SceneData normalizer 规则、运行时游戏入口、渲染器主路径改造、Gizmo、Undo/Redo、资源浏览器、Prefab、热重载或 Play Mode。
  - 允许依赖：`Engine.Editor`、`Engine.SceneData`、`Engine.Contracts`、`Engine.Platform` 或 OpenTK/ImGui 相关库。
  - 禁止：绕过 `SceneEditorSession` 直接改写 scene JSON 业务规则。

- `Engine.Editor`
  - 继续保持 headless core。
  - 禁止新增 GUI、OpenTK、ImGui、窗口或输入依赖。

- `Engine.App`
  - 继续作为运行时入口。
  - M13 不把编辑器模式塞进 `Engine.App`，也不改变默认运行路径。

## Milestones

- M13.1: Editor GUI 宿主入口
  - 交付结果：
    - 新增 `src/Engine.Editor.App/Engine.Editor.App.csproj`。
    - 新增 `.ai-workflow/boundaries/engine-editor-app.md`。
    - 接入 solution。
    - 启动后打开编辑器窗口，创建 `SceneEditorSession`，默认加载 sample scene 源文件。
  - 验收口径：
    - `dotnet run --project src/Engine.Editor.App/Engine.Editor.App.csproj` 能启动并关闭编辑器窗口。
    - `Engine.App` 运行路径不变。
    - `dotnet test AnsEngine.sln` 通过。

- M13.2: 基础布局与状态栏
  - 交付结果：
    - Toolbar：`Open`、`Save`、`Save As`、`Add Object`、`Remove Selected`。
    - Hierarchy 区域占位。
    - Inspector 区域占位。
    - Status Bar 显示当前 scene path、dirty 状态、selected object id、last error。
  - 验收口径：
    - 启动后布局稳定。
    - 默认 scene path 可见。
    - dirty 状态来自 `SceneEditorSession`。
    - 没有选中对象时 Inspector 显示空状态。

- M13.3: Hierarchy 面板与选择联动
  - 交付结果：
    - Hierarchy 读取 `SceneEditorSession.Objects`。
    - 每个 object 显示 object name 和 object id。
    - 点击对象调用 `SceneEditorSession.SelectObject(objectId)`。
    - 当前选中对象高亮。
  - 验收口径：
    - 默认场景对象能显示在 Hierarchy。
    - 点击对象后 `SelectedObjectId` 更新。
    - selection 变化同步到状态栏和 Inspector。
    - selection 不改变 dirty。

- M13.4: Inspector 对象编辑
  - 交付结果：
    - Inspector 显示并编辑选中对象的 `Id`、`Name`、`Mesh`、`Material`、`Position`、`Rotation`、`Scale`。
    - 所有修改都调用 `SceneEditorSession`：`UpdateObjectId`、`UpdateObjectName`、`UpdateObjectResources`、`UpdateObjectTransform`。
  - 验收口径：
    - 修改 name/transform 后 Hierarchy/Inspector 立即反映新值。
    - 成功编辑后 `IsDirty=true`。
    - 失败时显示 last error，UI 回到 session 当前有效值。
    - GUI 不直接修改 JSON DTO。

- M13.5: Open / Save / Save As 工作流
  - 交付结果：
    - GUI 接入 `Open`、`Save`、`SaveAs`。
    - 支持环境变量 `ANS_ENGINE_EDITOR_SCENE_PATH` 覆盖默认 scene。
    - 默认编辑源码目录下的 sample scene，而不是 bin 输出目录。
  - 验收口径：
    - 修改对象后 Save，文件真实写回。
    - Save 成功 `IsDirty=false`。
    - Save 失败 `IsDirty` 保持 true。
    - Open 失败不污染当前 session。
    - Save As 成功后状态栏路径更新。

- M13.6: Add / Remove Object GUI 工作流
  - 交付结果：
    - Add 创建默认 cube 对象并自动选中。
    - Remove Selected 删除当前选中对象。
    - 默认 id 使用 `object-001`、`object-002` 递增避重。
    - 默认 mesh 为 `mesh://cube`，默认 material 为 `material://default`，默认 transform 为 position `0,0,0`、identity rotation、scale `1,1,1`。
  - 验收口径：
    - Add 后 Hierarchy 出现新对象。
    - Add 后 Inspector 显示新对象。
    - Remove Selected 后对象消失，selection 清空。
    - 保存后 reload 仍能看到增删结果。

- M13.7: QA 收口与运行验收
  - 交付结果：
    - 自动测试与人工 smoke 完成。
    - 任务归档、计划归档索引和看板状态收口。
  - 验收口径：
    - `dotnet test AnsEngine.sln` 通过；只允许已有 `net7.0` EOL warning。
    - 人工 smoke：启动 Editor GUI -> 默认打开 sample scene -> 选择对象 -> 修改 name 或 transform -> Save -> 关闭 Editor -> 运行 `Engine.App` -> 确认运行时读取修改后的 scene。
    - `Engine.Editor` 仍无 GUI 依赖。
    - `Engine.App` 默认运行时入口不变。
    - `Engine.Render` 不感知 Editor GUI。

## PriorityOrder

- P0: M13.1
  - 原因：先建立独立 GUI 宿主入口和边界，避免把编辑器塞进 `Engine.App` 或污染 `Engine.Editor`。
- P1: M13.2
  - 原因：稳定基础布局和状态栏，让后续 Hierarchy/Inspector 有确定落点。
- P2: M13.3
  - 原因：Hierarchy selection 是 Inspector 编辑的前置。
- P3: M13.4
  - 原因：Inspector 是 M13 “人能编辑”的核心能力。
- P4: M13.5
  - 原因：Open/Save/Save As 让 GUI 编辑形成文件闭环。
- P5: M13.6
  - 原因：Add/Remove 是最小场景编辑器自然预期能力，但依赖已有布局、selection 和保存语义。
- P6: M13.7
  - 原因：QA 最后收口。

## Risks

- 高风险：GUI 逻辑绕过 `SceneEditorSession` 直接改 JSON。
  - 规避策略：所有编辑按钮和 Inspector 字段提交必须调用 M12 session API。
- 高风险：把 GUI 依赖塞进 `Engine.Editor`。
  - 规避策略：GUI 只存在于 `Engine.Editor.App`，并补边界测试。
- 中风险：OpenTK + ImGui.NET 接线复杂。
  - 规避策略：M13.1 先做最小窗口和主循环，后续逐步加 UI 面板。
- 中风险：默认路径误写 bin 输出目录。
  - 规避策略：默认解析源码目录 sample scene，并允许环境变量覆盖。
- 低风险：`net7.0` 已 EOL。
  - 规避策略：记录为独立技术债，不阻塞 M13。

## Assumptions

- 用户希望 M13 后能真正运行 GUI 编辑器，而不是继续只验证 API。
- M13 GUI 第一版接受工具型 ImGui 风格，不追求最终产品级布局系统。
- 文件选择可以先用路径输入或最小文件选择，不要求原生系统文件对话框。
- M13 不要求可视 viewport 或 3D 物体点击选择。
