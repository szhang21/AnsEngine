## GoalSummary

- 本轮目标：M12 不做 GUI 本身，而是为 M13 GUI 编辑器建立可直接消费的无界面编辑器核心。
- 成功标准（可验证）：
  - 新增 `Engine.Editor` 模块，承载场景编辑会话、dirty 状态、selection 状态、编辑命令编排、保存与 reload 验证。
  - `Engine.Editor` 消费 M11 已完成的 `Engine.SceneData` 文档读写、校验和对象级编辑能力，不重复实现 JSON 编辑规则。

- M12 的 selection 是纯逻辑状态，例如当前选中的 object id，不包含鼠标拾取、视口射线或渲染高亮。

## ModuleBoundaries

- `Engine.Editor`
  - 职责：编辑器会话、当前文档路径、当前文档快照、规范化场景快照、dirty 状态、selection 状态、编辑命令编排、保存与 reload 验证。
  - 非职责：GUI、窗口、输入事件、OpenGL、渲染高亮、资源导入、asset existence 校验、运行时场景对象、Undo/Redo。
  - 允许依赖：`Engine.SceneData`、`Engine.Contracts`、必要时 `Engine.Core`。
  - 禁止依赖：`Engine.App`、`Engine.Render`、`Engine.Platform`、`Engine.Asset`、OpenTK、OpenGL。

- `Engine.SceneData`
  - 继续负责文档 DTO、JSON 读写、校验、默认值规范化、对象级编辑原语。
  - 不承载 dirty、selection、打开中的 session 或 GUI 状态。

- `Engine.App`
  - M12 默认不接入 Editor，避免在 GUI 前提前改变主启动路径和组合根职责。

- `Engine.Render`
  - M12 不感知 Editor，不消费编辑器 session，不参与编辑状态。

    - `Engine.Editor` 不引用 App/Render/Platform/Asset。

- M12.2: 打开场景与会话状态
  - 交付结果：
    - 新增主入口类型，建议命名为 `SceneEditorSession`。
    - 支持打开 `.scene.json`、关闭当前文档、查询当前路径、文档、对象列表、dirty 状态和 selection 状态。
    - 打开时执行 normalize/reload 验证，确保当前文档可转换为 `SceneDescription`。
  - 验收口径：
    - 打开合法 sample scene 成功，`IsDirty=false`，`SelectedObjectId=null`。
    - 打开缺失文件、非法 JSON、非法 scene 均返回显式失败。
    - 打开失败不污染已有 session。
    - 关闭当前文档会清空路径、文档、selection，并重置 dirty。

- M12.3: 编辑命令与 selection/dirty 语义
  - 交付结果：
    - Editor session 包装 M11 的 `SceneFileDocumentEditor`。
    - 支持新增对象、删除对象、删除当前选中对象、修改对象 id/name、修改 mesh/material、修改 transform。
    - 支持选择对象与清空选择。
    - 成功编辑更新当前文档和规范化场景快照，并设置 `IsDirty=true`。
  - 验收口径：
    - 无文档时 select/edit/save 返回 `NoDocumentOpen` 或等价显式失败。
    - 选择存在对象成功且不改变 dirty。
    - 选择不存在对象失败且 selection 不变。
    - 编辑失败不改变文档、selection 或 dirty。
    - 删除当前选中对象后 selection 清空。
    - 修改当前选中对象 id 后 selection 跟随新 id。
    - 重复 id、空 mesh、非法 material、非有限 transform 都返回明确失败。

- M12.4: 保存、另存为与 reload 验证
  - 交付结果：
    - 支持保存当前文档到当前路径。
    - 支持另存为新路径。
    - 保存后通过 `JsonSceneDescriptionLoader` 或 normalizer 重新验证保存后的文件。
  - 验收口径：
    - `open -> edit -> save -> reload` 成功。
    - 保存成功后 `IsDirty=false`。
    - 保存失败后内存文档保留，`IsDirty` 保持 true。
    - 另存为成功后 session 当前路径切换为新路径，`IsDirty=false`。
    - 保存后的文件可被现有运行时 loader 加载为 `SceneDescription`。

## PlanningDecisions

- 默认方向：M12 做 GUI 编辑器的 headless core，M13 再做真实 GUI。
- 模块命名：默认使用 `Engine.Editor`，而不是 `Engine.Tools`。
- 主入口：默认使用 session 风格入口，建议命名 `SceneEditorSession`。
- 结果语义：所有公开操作返回显式结果类型，不用 bool/null 表达失败。
- Dirty 语义：只有成功编辑或保存失败后的未保存内存修改会保持 dirty；选择对象不改变 dirty。
- Selection 语义：selection 只保存 object id，不保存 UI 状态、鼠标状态或渲染高亮状态。
- 保存语义：保存成功必须 reload/normalize 验证；保存失败不得丢失内存修改。
- 关闭语义：M12 不做未保存关闭确认；GUI 确认流程留给 M13。
- 资源校验：M12 不校验 mesh/material 资源是否真实存在，只沿用 SceneData 的引用格式校验。
- App 接入：M12 不改变 `Engine.App` 默认启动路径。

## SuggestedPublicInterfaces

- `SceneEditorSession`
  - 查询：`HasDocument`、`SceneFilePath`、`IsDirty`、`SelectedObjectId`、`Document`、`Scene`、`Objects`、`SelectedObject`。
  - 操作：`Open`、`Close`、`SelectObject`、`ClearSelection`、`AddObject`、`RemoveObject`、`RemoveSelectedObject`、`UpdateObjectId`、`UpdateObjectName`、`UpdateObjectResources`、`UpdateObjectTransform`、`Save`、`SaveAs`、`ReloadValidate`。

- `SceneEditorSessionResult`
  - 表达操作成功/失败。
  - 成功时可返回 session 快照或更新后的核心状态。
  - 失败时返回 `SceneEditorFailure`。

- `SceneEditorFailure`
  - 包含 failure kind、message，可选 path/object id 等诊断字段。

- `SceneEditorFailureKind`
  - 建议覆盖：`None`、`NoDocumentOpen`、`OpenFailed`、`SaveFailed`、`ReloadValidationFailed`、`ObjectNotFound`、`DuplicateObjectId`、`InvalidReference`、`InvalidTransform`、`InvalidDocument`、`SelectionInvalid`。

## Risks

- 高风险：Editor 模块变成 GUI 或工具大杂烩。
  - 规避策略：M12 只做 session 和命令编排，不做窗口、面板、输入或渲染交互。
- 中风险：Editor 与 SceneData 职责重叠。
  - 规避策略：SceneData 保持文档原子操作；Editor 只组织打开中的会话状态和工作流。
- 中风险：保存失败导致内存修改丢失。
  - 规避策略：保存失败不得替换当前文档，不得清 dirty。
- 中风险：M13 GUI 需要未保存关闭确认。
  - 规避策略：M12 只暴露 `IsDirty`，确认流程留给 M13 GUI。
- 低风险：当前目标框架 `net7.0` 已 EOL。
  - 规避策略：记录为独立技术债，不混入 M12 功能范围。

## Assumptions

- M13 将推进 GUI 编辑器，所以 M12 优先服务 GUI，而不是做长期 CLI 产品。
- M12 不实现 Undo/Redo；M13 GUI 可以先没有撤销能力。
- M12 不做 asset existence 校验；真实资源存在性仍属于 Asset/运行链路问题。
- M12 不修改 `Engine.App` 主启动路径。
- M12 的选中对象是逻辑 id 状态，不包含鼠标拾取、视口射线或渲染高亮。
