# Engine.Asset 模块边界合同

## 1) 模块信息

- 模块名：`Engine.Asset`
- 版本：`v1.0`
- 负责人：`待指定`
- 生效日期：`2026-04-04`
- 关联任务卡：`待关联`

## 2) 目标与范围

- 模块目标：负责资源加载、缓存、重载与资源句柄管理，为上层模块提供稳定资源访问能力。
- 适用范围：纹理/网格/着色器文件读取、解析、缓存、生命周期管理。
- 非适用范围：场景图变更、渲染提交、窗口输入处理、应用流程编排。

## 3) 职责（Responsibilities）

- 负责资源加载管线（读取、解析、构建资源对象）。
- 负责资源缓存与去重策略。
- 负责资源句柄分配、查找与释放。
- 负责资源加载错误与版本冲突日志。

## 4) 非职责（Non-Responsibilities）

- 不负责场景实体挂接与运行态拓扑修改（归属 `Engine.Scene`）。
- 不负责 OpenGL 绘制流程与状态切换（归属 `Engine.Render`）。
- 不负责输入设备事件处理（归属 `Engine.Platform`）。
- 不负责主流程调度（归属 `Engine.App`）。

## 5) 允许依赖（Allowed Dependencies）

- 可直接依赖模块：
  - `Engine.Core`
  - `Engine.Platform`（仅文件系统抽象）
- 可使用基础库/第三方：
  - `AssimpNet`（模型导入）
  - `ImageSharp` / `StbImageSharp`（图像读取）

## 6) 禁止依赖（Forbidden Dependencies）

- 禁止直接依赖模块：
  - `Engine.Scene` 内部实体实现
  - `Engine.Render` 渲染队列实现细节
- 禁止跨层调用模式：
  - 在 `Asset` 内直接操作场景对象
  - 在 `Asset` 内直接驱动渲染帧循环

## 7) 公开接口（Public Interfaces）

- `IAssetService`
  - 用途：统一资源加载与查找入口
  - 输入/输出：输入资源标识（路径/键），输出资源句柄或加载结果
  - 错误语义：可恢复失败返回结果状态；不可恢复错误抛异常
  - 生命周期约束：初始化后加载，关闭时统一释放可管理资源

- `IAssetHandle`
  - 用途：上层安全引用资源对象
  - 输入/输出：输入句柄请求，输出资源引用或状态
  - 错误语义：失效句柄必须可检测
  - 生命周期约束：遵循引用有效期与释放策略

## 8) 数据与状态边界

- 模块内部可变状态：资源缓存表、句柄映射、重载标记。
- 外部可观察状态：加载状态、命中率统计、失败日志。
- 线程模型与并发约束：加载可异步，缓存写入需加锁或串行化。
- 资源生命周期：`Load -> Cache -> Use -> Release`，禁止悬空句柄。

## 9) 质量门禁与验收

- Build 验收：`dotnet build -c Debug/Release` 通过。
- Test 验收：加载、缓存、句柄有效性测试通过。
- Smoke 验收：Demo 场景资源加载稳定、无明显异常。
- Perf 验收：缓存命中与加载耗时满足基线预期。
- 边界验证项（必须逐条通过）：
  - [ ] 无禁止依赖
  - [ ] 无越界职责实现
  - [ ] 公开接口与合同一致
