# Engine.Platform 模块边界合同

## 1) 模块信息

- 模块名：`Engine.Platform`
- 版本：`v1.0`
- 负责人：`待指定`
- 生效日期：`2026-04-04`
- 关联任务卡：`待关联`

## 2) 目标与范围

- 模块目标：封装平台相关能力（窗口、输入、时间、文件系统桥接），为上层提供统一运行环境抽象。
- 适用范围：窗口生命周期、输入事件转发、时间步长、基础平台服务适配。
- 非适用范围：渲染策略、场景数据管理、资源导入逻辑、应用业务流程策略。

## 3) 职责（Responsibilities）

- 负责窗口创建、更新与关闭事件。
- 负责输入状态采集与标准化事件输出。
- 负责时间与帧间隔数据提供。
- 负责文件系统基础访问抽象（如路径与读写能力接口）。

## 4) 非职责（Non-Responsibilities）

- 不负责渲染管线与 GPU 指令提交（归属 `Engine.Render`）。
- 不负责场景对象管理（归属 `Engine.Scene`）。
- 不负责资源解析与缓存策略（归属 `Engine.Asset`）。
- 不负责模块编排与业务启动策略（归属 `Engine.App`）。

## 5) 允许依赖（Allowed Dependencies）

- 可直接依赖模块：
  - `Engine.Core`
- 可使用基础库/第三方：
  - `OpenTK`（窗口与输入）
  - `System.IO`（文件系统基础能力）

## 6) 禁止依赖（Forbidden Dependencies）

- 禁止直接依赖模块：
  - `Engine.Scene` 内部实现
  - `Engine.Asset` 导入器实现
- 禁止跨层调用模式：
  - 在平台层直接执行渲染策略决策
  - 在平台层直接修改场景运行态

## 7) 公开接口（Public Interfaces）

- `IWindowService`
  - 用途：窗口生命周期和基础属性管理
  - 输入/输出：输入窗口配置，输出窗口状态、关闭意图与事件轮询能力
  - 错误语义：初始化失败抛异常；运行时异常需记录并上抛
  - 生命周期约束：应用启动阶段创建，循环阶段调用 `ProcessEvents`，退出阶段销毁

- `IInputService`
  - 用途：统一输入查询和事件分发
  - 输入/输出：输入平台输入源，输出标准化输入状态
  - 错误语义：设备不可用需返回可诊断状态
  - 生命周期约束：随窗口服务启停

- `ITimeService`
  - 用途：提供 delta time、累计时间、帧率统计
  - 输入/输出：输入时钟源，输出时间快照
  - 错误语义：时间异常需显式告警
  - 生命周期约束：应用生命周期内持续有效

## 8) 数据与状态边界

- 模块内部可变状态：窗口句柄、输入快照、时间状态缓存。
- 外部可观察状态：窗口状态、输入事件流、帧时间信息。
- 线程模型与并发约束：窗口与输入主线程优先；跨线程读取需只读快照。
- 资源生命周期：平台资源随应用启动创建、退出释放，禁止悬挂句柄。

## 9) 质量门禁与验收

- Build 验收：`dotnet build -c Debug/Release` 通过。
- Test 验收：输入映射、时间服务、窗口生命周期相关测试通过。
- Smoke 验收：窗口与输入在 Demo 场景稳定可用。
- Perf 验收：平台层每帧开销稳定，无异常尖峰。
- 边界验证项（必须逐条通过）：
  - [ ] 无禁止依赖
  - [ ] 无越界职责实现
  - [ ] 公开接口与合同一致
