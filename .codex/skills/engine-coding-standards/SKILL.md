# Engine 编码规范

当需要统一项目内的 C# 编码风格时，使用本 Skill。

## 命名规则

- `private` 与 `protected` 字段使用 `camelCase`，禁止前导下划线。
- 构造器参数、局部变量、方法参数使用 `camelCase`。
- 公共类型、公共属性、公共方法使用 `PascalCase`。
- 常量使用 `PascalCase` 或按项目既有约定保持一致，但不要混用前导下划线字段风格。

## 迁移规则

- 如果需要从 `_fieldName` 迁移到 `fieldName`，必须同步更新：
  - 字段声明
  - 构造器赋值
  - 所有成员访问点
  - 反射测试中的字段名字符串
- 迁移过程中不得保留“双写”命名，避免同一类中同时出现旧式和新式字段。

## 约束

- 新增代码默认不得再引入前导下划线私有字段。
- 代码审查时如果发现旧式字段名，应优先以局部重命名方式修正，而不是沿用旧风格。
