# Engine 编码规范

当需要统一项目内的 C# 编码风格时，使用本 Skill。

## 命名规则

- `private` 与 `protected` 实例字段使用 `mCamelCase`，即前缀 `m` + 小驼峰，禁止前导下划线。
- 静态字段使用 `sCamelCase`，即前缀 `s` + 小驼峰。
- `const` 常量使用 `kCamelCase`，即前缀 `k` + 小驼峰。
- 构造器参数、局部变量、方法参数使用 `camelCase`。
- 公共类型、公共属性、公共方法使用 `PascalCase`。
- 同一个类型里不要同时保留下划线字段和新风格字段。

## 文件组织规则

- 默认一个类一个文件，一个接口一个文件。
- 一个文件默认只放一个主类型。
- 只有在以下情况允许例外：
  - 很小且紧密相关的私有辅助类型或 record。
  - 嵌套类型本来就是主类型的实现细节。
  - 测试桩、局部 helper、迁移过渡期的临时共存。
- 如果例外成立，优先保证可读性和边界清晰，不要为了“少文件”把职责混在一起。

## 迁移规则

- 如果需要把 `_fieldName` 迁移到 `mFieldName`，必须同步更新：
  - 字段声明
  - 构造器赋值
  - 所有成员访问点
  - 反射测试中的字段名字符串
- 如果需要把普通静态字段迁移到 `sFieldName`，必须同步更新：
  - 静态字段声明
  - 所有访问点
  - 反射测试中的字段名字符串
- 如果需要把普通常量迁移到 `kValueName`，必须同步更新：
  - 常量声明
  - 所有访问点
  - 反射测试中的字段名字符串

## 约束

- 新增代码默认不再引入前导下划线私有字段。
- 新增代码默认不使用无前缀的实例字段、静态字段、常量命名。
- 代码审查时如发现旧风格字段命名，应优先做局部重命名修正，而不是继续扩散旧风格。
