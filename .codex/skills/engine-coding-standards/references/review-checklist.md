# 评审检查清单

在合并前或发起 AI 自评前，请按本清单逐条检查。

## 1) 范围与边界

- [ ] 实际改动范围仍与原始需求一致，无隐藏扩展。
- [ ] 模块边界被遵守。
- [ ] 未引入禁止方向的依赖关系。
- [ ] 公开 API 变更是有意为之，且已记录原因。

## 2) 正确性

- [ ] 主逻辑路径至少有一项可执行验证覆盖。
- [ ] 异常路径已处理（或明确记录为超出本次范围）。
- [ ] 未出现静默吞异常。
- [ ] 资源生命周期完整（`create/use/dispose`）。

## 3) 性能（引擎热路径）

- [ ] Update/Render 热路径没有可避免的逐帧分配。
- [ ] 热路径未引入可避免的 LINQ/装箱。
- [ ] 可重复查询已按需缓存。
- [ ] 渲染状态切换已最小化且无冗余。

## 4) 可读性与可维护性

- [ ] 命名清晰且贴合领域语义。
- [ ] 文件路径与命名空间一致。
- [ ] 公开契约简洁明确。
- [ ] 注释解释“非显而易见的意图”，而非重复代码字面含义。
- [ ] 类职责单一，没有把渲染/资源/流程编排等多类职责揉进同一类型（S）。
- [ ] 未引入“胖接口”；高层模块依赖抽象而非底层实现细节（I/D）。

## 5) 验证门禁

- [ ] `dotnet build -c Debug` 通过。
- [ ] `dotnet build -c Release` 通过。
- [ ] 受影响模块 `dotnet test` 通过。
- [ ] Demo 场景冒烟运行稳定。
- [ ] 相比基线无明显帧时间退化。

## 6) 评审输出完整性

- [ ] `Scope` 用一句话给出。
- [ ] `BoundaryCheck` 明确写出 `pass/fail` 及违规项。
- [ ] `CorrectnessRisks` 含风险等级。
- [ ] `PerformanceRisks` 含风险等级。
- [ ] `MaintainabilityRisks` 含风险等级。
- [ ] `MustFixBeforeMerge` 与 `OptionalImprovements` 已分开。
- [ ] `Validation` 已区分“已执行”和“未执行”。

## 7) 最终结论

- [ ] 可合并（Ready to merge）。
- [ ] 阻塞（Blocked），并附阻塞项与责任人。
